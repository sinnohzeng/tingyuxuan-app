# 听语轩 产品需求文档（PRD）

> **版本**: v3.0 | **日期**: 2026-03-03 | **当前产品版本**: v0.10.3
>
> **产品定位**：AI 驱动的智能语音输入工具 — 将口语化的语音自动转为规范的书面文字。
>
> **产品代号**：听语轩（TingYuXuan / Listencraft）
>
> **对标产品**：[Typeless](https://www.typeless.com/)（详见 [竞品分析](competitive-analysis.md)）

---

## 一、产品概要

### 1.1 一句话定义

听语轩 = 多模态 LLM 一步语音识别与润色 + 系统级全局文本注入。

用户在任何应用中按下快捷键，对着麦克风说话，听语轩通过多模态大语言模型**一步完成**语音识别、去口语化、智能排版，然后将润色后的文本自动输入到光标位置。

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| **速度** | 语音输入 ~220 WPM，是键盘打字（~45 WPM）的近 5 倍 |
| **质量** | 自动去填充词、修正口误、整理排版，输出即发送级文本 |
| **全局性** | 系统级文本注入，在任何应用中均可使用 |
| **隐私** | 用户自带 API Key，零数据留存，本地处理音频 |
| **简单** | 仅需配置一个 API Key，无需独立的 STT 服务 |

### 1.3 产品命名

- **听**：聆听 — 产品的核心输入方式
- **语**：语言 — 连接口语与书面语的桥梁
- **轩**：高雅、开阔 — 寓意将粗糙的口语提升为精致的文字

### 1.4 目标用户

- **知识工作者**：需要大量文字输出的职场人士（邮件、报告、文档）
- **多语言工作者**：需要跨语言沟通的用户（翻译模式）
- **效率极客**：追求输入速度极致的用户
- **无障碍需求者**：因身体原因难以长时间使用键盘的用户

## 三、功能需求

### 3.1 听写模式（Dictate）— 默认模式 ✅ 已实现

将口语化的语音转为规范的书面文字。

**工作流**：按下快捷键 → 对着麦克风说话 → 再按快捷键停止 → AI 一步识别+润色 → 文本注入光标位置。

**智能处理能力**：
- 去除填充词（嗯、啊、那个、um、uh、like 等）
- 检测并修正口误（用户改口时只保留最终意图）
- 消除连续重复（"这个这个这个方案" → "这个方案"）
- 自动补充标点符号，智能断句断段
- 序号/要点检测，自动生成有序/无序列表
- 参考用户词典提高专有名词准确率

**上下文感知语气调整**：

| 应用类型 | 语气 |
|----------|------|
| 即时通讯（微信、Slack） | 口语化、简短 |
| 邮件（Outlook、Gmail） | 正式、完整句子 |
| 代码编辑器（VS Code） | 保留技术术语 |
| 笔记应用（Notion） | 结构化、Markdown |

> 详细 prompt 规格见 [多模态 LLM 模块](modules/llm.md)

### 3.2 翻译模式（Translate）✅ 已实现

将语音直接翻译为目标语言。输出不是直译，而是本地化的自然表达。

**工作流**：按下翻译快捷键 → 用任意语言说话 → 停止 → AI 一步识别+翻译 → 翻译结果注入光标位置。

**翻译质量要求**：
- 意译优于直译，追求目标语言的自然表达
- 保持原文语气和意图
- 支持混合语言输入（如中英混说 → 纯目标语言输出）
- 自动应用与听写模式相同的去口语化处理

### 3.3 AI 助手模式（AI Assistant）✅ 已实现

将语音作为自由形式的 AI 指令。

**工作流**：按下 AI 助手快捷键 → 说出请求 → 停止 → AI 执行指令 → 结果显示在结果面板中（不自动注入）。

**与听写模式的区别**：

| 维度 | 听写模式 | AI 助手模式 |
|------|----------|------------|
| 输入 | 用户想表达的内容 | 给 AI 的指令/问题 |
| 输出 | 用户话语的优化版 | AI 生成的全新内容 |
| 注入方式 | 自动插入光标 | 结果面板（手动复制/插入） |

**结果面板特性**：Markdown 渲染、一键复制、插入到光标。

### 3.4 语音编辑模式（Speak to Edit）⏳ 部分实现

用语音指令修改已选中的文本。

**工作流**：选中文本 → 按下快捷键 → 说出编辑指令 → AI 修改选中文本。

**支持的编辑操作**：

| 操作类型 | 示例指令 |
|----------|---------|
| 语气调整 | "让这段更正式一些" |
| 格式转换 | "把这段变成列表" |
| 内容修改 | "把日期改成下周五" |
| 扩写/缩写 | "精简到一句话" |
| 翻译 | "翻译成英文" |
| 纠错 | "修正拼写错误" |

> 当前依赖上下文增强方案（UIA/OCR）完整落地，详见 [v0.10.3 技术债修复计划](plan/2026-03-03-technical-debt-remediation-v0.10.3.md)。

### 3.5 个人词典 ✅ 已实现

用户可手动添加专有名词、品牌名、行业术语，提高识别和处理准确率。

- 设置 → 词典 → 添加/删除词汇
- 每个词汇最长 100 字节
- 词典内容在 LLM prompt 中作为参考

### 3.6 历史记录与仪表盘 ✅ 已实现

- **历史记录**：分页浏览、关键词搜索、单条/批量删除、一键清空
- **仪表盘**：总录音时长、总字数、录音次数、平均速度统计
- 数据存储在本地 SQLite，默认保留 30 天

## 四、交互设计规格

### 4.1 浮动状态条

核心 UI 组件，录音和处理过程中悬浮在屏幕底部。

**三栏布局**：

```
┌──────────────────────────────────────────────────────┐
│  ✕  │          ▮▮▮▮▮ 波形可视化 ▮▮▮▮▮            │  ■  │
│ 取消 │          （实时声纹条形图）                    │ 停止 │
└──────────────────────────────────────────────────────┘
```

**状态机（6 态）**：

```
idle ──→ starting ──→ recording ──→ thinking ──→ done
  ▲                      │                        │
  │                    cancel                     │
  │                      │                        │
  └─── cancelled ◄──────┘    error ◄─── (失败) ──┘
  └──────────────────────────────────────────────┘
```

| 状态 | 浮动条表现 | 持续时间 |
|------|-----------|---------|
| idle | 隐藏 | — |
| starting | 微加载动画 | ~50ms |
| recording | 模式标签 + 实时波形 + 停止/取消按钮 | 用户控制（≤5 分钟） |
| thinking | "思考中…" 加载动画 | ~1-3 秒 |
| done | 成功提示 | 1.5 秒后自动隐藏 |
| error | 错误信息 + 操作建议 | 用户手动关闭 |

**波形可视化**：~20-40 根垂直条形图，随实时音量变化，≥30fps 刷新。

**录音限制**：第 4 分钟起显示 60 秒倒计时警告，5 分钟自动停止进入 thinking。

### 4.2 快捷键体系

| 功能 | macOS | Windows / Linux | 说明 |
|------|-------|-----------------|------|
| 听写 | `Fn` | `RAlt` | 按下开始，再按停止 |
| 翻译 | `⌥T` | `Shift+RAlt` | 启动翻译模式 |
| AI 助手 | `⌃Space` | `Alt+Space` | 启动 AI 助手 |
| 取消 | `Esc` | `Esc` | 取消当前录音 |

**设计原则**：所有快捷键可在设置中自定义。macOS 基于 Fn 键，Windows/Linux 基于右 Alt 键。

> 详细快捷键说明见 [使用指南](guides/usage.md)

### 4.3 系统托盘

托盘菜单结构：反馈 → 设置 → 麦克风选择 → 词典 → 版本/更新 → 退出。

> 详细菜单规格见 [UI 设计规格](architecture/ui-design.md)

### 4.4 设置面板

| 分类 | 设置项 |
|------|--------|
| API 配置 | Provider 选择、API Key、Base URL、连接测试 |
| 语言 | 主语言、翻译目标语言、变体 |
| 快捷键 | 四种模式快捷键自定义 |
| 通用 | 开机自启、声音反馈、浮动条位置、最小化到托盘 |
| 词典 | 添加/删除自定义词汇 |
| 麦克风 | 输入设备选择 |

> 注意：当前 MVP 阶段模型由程序固定（qwen3-omni-flash），不开放用户切换。
>
> 详细配置说明见 [配置指南](guides/configuration.md)

首次启动三步引导：产品介绍（3 屏）→ API 配置向导 → 权限检查 → 进入首页。

## 五、异常处理与数据安全

### 5.1 核心原则

> **用户说的每一个字都不能丢。** 无论网络中断、服务崩溃还是应用闪退，用户的录音内容都必须有迹可循、可恢复。

### 5.2 异常场景

| 场景 | 用户体验 | 恢复策略 |
|------|----------|----------|
| 网络失败 / API 超时 | 浮动条显示错误 + 可操作建议 | 自动重试 1 次；失败后提供重试按钮 |
| 麦克风不可用 / 无权限 | 系统通知提示 + 跳转设置链接 | 不显示浮动条，避免误以为在录音 |
| 超长录音（>5 分钟） | 第 4 分钟起倒计时警告 | 5 分钟自动截止，立即进入处理 |
| API Key 未配置 / 无效 | 连接测试失败提示 | 引导用户进入设置页配置 |
| 模型返回无效结果 | 质量闸门拦截 + 错误提示 | 占位文本/模板输出直接报错 |

### 5.3 数据安全

- **API Key**：存储在操作系统 keyring（非明文），配置文件仅存引用标记
- **音频数据**：内存缓冲区处理，不写入磁盘文件
- **历史记录**：仅存文本，本地 SQLite，支持一键清空
- **网络请求**：所有 API 调用在本地 Rust 后端发起，零第三方中转

## 八、约束与风险

### 8.1 技术约束

| 约束 | 说明 |
|------|------|
| 多模态模型 | 所选模型必须支持音频输入，纯文本模型不可用 |
| 录音时长 | 单次 ≤ 5 分钟（模型上下文窗口限制） |
| API 超时 | 60 秒（音频处理比纯文本慢） |
| 最短录音 | ≥ 300ms（防误触） |

### 8.2 已知风险

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| 固定模型下线 | 服务完全不可用 | 模型常量可快速回退；支持多 Provider |
| 快捷键系统冲突 | Alt+Space 被 Windows 拦截 | 所有快捷键可自定义 |
| 输入法兼容性 | 中文输入法下可能异常 | 长文本自动切换剪贴板粘贴 |
| MP3 编码失败 | 上传体积增大 | 自动回退 WAV 格式 |
